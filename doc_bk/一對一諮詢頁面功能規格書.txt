一對一諮詢頁面功能規格書

1. 頁面佈局與結構 (Page Structure)

1.1 導師個人檔案區 (Consultant Profile)

核心資訊：

導師頭像、姓名、專業職稱。

專業領域標籤 (Tags)：例如「職涯規劃」、「前端開發」、「面試技巧」。

諮詢統計：總諮詢次數、平均評分、回覆速度。

個人簡介：詳細的學經歷、專業背景及能提供的幫助說明。

1.2 諮詢方案選擇 (Service Options)

方案清單：

顯示不同的諮詢時長（例如：30 分鐘、60 分鐘）。

各方案對應的價格。

諮詢方式說明（例如：Google Meet 視訊、語音通話）。

1.3 預約時段選取器 (Time Slot Picker)

日曆視圖 (Calendar View)：顯示可預約的日期。

時段列表 (Time Slots)：

點擊日期後顯示當天可供預約的具體時段。

時區切換：自動偵測使用者時區，並提供手動調整功能。

即時狀態：已預約或已過期的時段應顯示為禁用 (Disabled)。

1.4 諮詢需求表單 (Requirement Form)

前置資訊收集：

諮詢主題/具體問題描述（必填）。

相關附件上傳（例如：作品集連結、簡歷 PDF）。

預期目標說明。

2. API 整合邏輯 (API Integration)

2.1 獲取導師與時段資訊

API：GET /api/v1/consultants/{id}

行為：進入頁面時載入導師基本資料。

API：GET /api/v1/consultants/{id}/availability?start_date={date}&end_date={date}

行為：根據日曆切換的月份或週別，動態獲取可預約時段。

2.2 預約與訂單建立

API：POST /api/v1/consultations/book

行為：

提交選取的時段 ID、方案 ID 及需求表單內容。

成功後：後端建立諮詢紀錄並回傳 order_id。

跳轉至支付流程：POST /api/v1/orders/{id}/pay。

2.3 預約通知

行為：付款成功後，系統自動發送預約確認通知（Email/站內信）給導師與學員，包含視訊會議連結。

3. 核心功能邏輯 (Core Logic)

3.1 預約鎖定機制 (Locking)

行為：使用者選取時段並進入結帳流程時，該時段應暫時鎖定 (Lock) 10-15 分鐘，避免重複預約。若超時未付款則釋放。

3.2 視訊會議整合

行為：系統應自動串接第三方會議工具（如 Zoom 或 Google Meet），在預約成功後生成專屬連結。

3.3 評價與回饋

行為：諮詢結束後，系統發送評價邀請，學員可對導師進行評分與文字回饋。

4. 異常處理與狀態 (State & Error Handling)

時段衝突：若兩個使用者同時選取同一時段，後提交者應收到「該時段已被預約」的錯誤提示。

導師休假/停權：若導師狀態為不可服務，應顯示「目前不提供預約」並隱藏時段選取器。

退費與取消：

若在諮詢前 X 小時取消，可申請全額退費。

若導師無故缺席，學員可檢舉並申請退費。

5. 交互行為 (Interactions)

Sticky Sidebar (桌面端)：將預約摘要與「立即預約」按鈕固定在右側隨頁面捲動。

即時計算：根據選取的方案與優惠券，即時更新總金額。

行動端適配：時段選取器在手機上應改為底部彈出式面板 (Bottom Sheet) 以方便操作。
